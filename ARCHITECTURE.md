# Architecture

The system runs as two processes that communicate exclusively through Twilio's REST API. `caller.py` is the orchestrator: it starts an ngrok HTTPS tunnel once, launches the FastAPI server as a subprocess, then places Twilio outbound calls one at a time and polls Twilio's status API until each call reaches a terminal state. `main.py` is an async FastAPI server running a TwiML state machine. When a call connects, `/call/start` returns `<Connect><Stream>` TwiML to open a WebSocket, which forwards mulaw audio chunks to Deepgram's real-time STT. Two redundant silence detectors — Deepgram's `utterance_end` event (1.2s threshold) and a local asyncio watcher (1.5s) — both call `_trigger_redirect()`, which is serialized by an `asyncio.Lock` to prevent race conditions. The redirect updates the live call via Twilio's REST API to hit `/call/respond`, where the patient's next line is generated by Groq (`llama-3.1-8b-instant`) and spoken with `<Say voice="Polly.Matthew-Neural">`. After the call ends, a second Groq pass analyzes the full transcript for bugs and writes `result.json` and `analysis.txt`.

The key design choices were driven by Twilio's constraints. The only way to break out of an active `<Connect><Stream>` and inject new speech is `call.update(url=...)` via the REST API — there is no in-band signal you can send over the WebSocket to interrupt the stream. The dual silence detector exists because Deepgram's `utterance_end` can miss low-energy pauses (mumbling, background noise), so the local watcher is the safety net. The Groq call is pre-started at the moment of silence detection, before Twilio has even requested `/call/respond`, so the LLM latency is hidden behind Twilio's own redirect round-trip — keeping response time under 2 seconds. All 15 patient personas use the real identity James Bond (DOB June 14th 2002), which matches an actual record in the target agent's system; earlier runs with fictional personas spent 60–80% of every call stuck in identity verification, never reaching the scenario goal.
